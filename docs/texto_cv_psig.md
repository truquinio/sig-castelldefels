# Cómo presentarlo sin exagerar

## Línea para el CV

**Mini-proyecto SIG personal | GeoJSON, Leaflet, OpenStreetMap, ICGC, PostGIS**

Desarrollo de una visualización web de actividades y servicios urbanos en Castelldefels con datos abiertos. Integración de límite municipal ICGC y puntos OpenStreetMap/Overpass, filtrado espacial, clasificación por categorías, malla de 500 m, mapa interactivo en Leaflet y consultas PostGIS básicas para conteos e intersecciones.

## Versión corta para LinkedIn o GitHub

Mini-proyecto SIG de aprendizaje aplicado: mapa web de Castelldefels con límite municipal ICGC, puntos OSM/Overpass y análisis básico por categorías y malla de 500 m. El objetivo es demostrar un flujo completo y reproducible: obtención de datos, tratamiento GeoJSON, visualización Leaflet y consultas PostGIS.

## Texto para enviar a PSIG

Hola,

Estoy reforzando mi perfil hacia SIG y datos territoriales, conectándolo con mi experiencia actual en urbanismo y gestión de información municipal. Para mostrarlo de forma concreta preparé un mini-proyecto personal sobre Castelldefels: descarga de límite municipal ICGC, consulta de puntos OpenStreetMap con Overpass, filtrado espacial, malla de 500 m, visualización en Leaflet y consultas PostGIS básicas.

No lo presento como un censo oficial ni como un proyecto profesional cerrado, sino como una muestra sencilla y reproducible de mi forma de trabajar con datos geográficos abiertos.

## Qué conviene decir en entrevista

- "Es un proyecto personal para demostrar flujo SIG completo con datos abiertos."
- "No sustituye un inventario municipal validado."
- "Usé OSM como fuente abierta, pero documenté sus limitaciones."
- "La parte que más me interesó fue conectar datos territoriales, visualización web y consultas espaciales."
- "Me gustaría evolucionarlo con QGIS, PostGIS y una fuente oficial si estuviera disponible."

## Qué evitar

- No decir que es un censo de actividades económicas.
- No decir que los datos están validados por el Ayuntamiento.
- No decir que eres experto en PostGIS por este proyecto.
- No decir que el análisis mide actividad económica real.
- No llamar "barrios" a zonas que no provienen de una fuente oficial de barrios.

## Encaje con tu CV

Este proyecto refuerza tres partes reales de tu perfil:

- Urbanismo y gestión municipal: trabajas con expedientes, territorio y sistemas administrativos.
- Datos y desarrollo: ya tienes base en SQL, JavaScript, APIs, Git y automatización.
- SIG en crecimiento: el proyecto muestra práctica real con geodatos, sin fingir años de experiencia.
